# Event System: The Narrative Engine

## Overview

Events are the primary way the game tells its story and presents choices to players. Every significant moment - from protests erupting to secret negotiations to personal crises - is modeled as an event. The Event System determines what happens, when, and what choices players have.

---

## Event Philosophy

### Events as Teaching Moments

Every event should teach something:
- About how politics works
- About human psychology
- About the specific Iranian context
- About the consequences of choices

**Bad event:** "Protests erupt. What do you do?"
**Good event:** "Protests erupt in Isfahan after a video shows police beating a student. The student's family is calling for justice. Local merchants are closing shops in solidarity. The national media is silent, but Twitter is exploding. Regional IRGC commander is requesting permission to 'restore order.' Your intelligence says the protests could spread - or could fizzle in a day. What do you do?"

The good event teaches: how protests spread, the role of trigger events, media dynamics, regime decision-making, uncertainty.

### Events as Character Revelation

Events reveal who characters are through their choices:
- What does a player choose when there's no good option?
- How do they balance competing values?
- Who do they sacrifice for whom?

### Events as Historical Echoes

Events should connect to real patterns:
- "This situation resembles..."
- Learn More links to historical cases
- Players should recognize patterns across playthroughs

## Event Types

### 1. Story Events

Pre-authored events that advance major narrative arcs.

```yaml
STORY_EVENT:
  type: story
  characteristics:
    - Carefully crafted narrative
    - Major consequences
    - Often character-defining moments
    - Limited randomness
  examples:
    - Supreme Leader health crisis
    - Major protest waves
    - International interventions
    - Personal betrayals/alliances
```

### 2. Triggered Events

Events that fire when conditions are met.

```yaml
TRIGGERED_EVENT:
  type: triggered
  characteristics:
    - Condition-based activation
    - Consequences of previous choices
    - Faction actions in response to situation
    - System state changes
  examples:
    - "If economic_crisis > 80: trigger famine_concerns"
    - "If player_allied_with(monarchists) AND player_allied_with(republicans): trigger coalition_tensions"
    - "If character_trust(IRGC_General) > 70: unlock defection_conversation"
```

### 3. Scheduled Events

Events tied to the calendar.

```yaml
SCHEDULED_EVENT:
  type: scheduled
  characteristics:
    - Known dates (real Iranian calendar)
    - Annual commemorations
    - Political calendar (elections)
    - Religious calendar
  examples:
    - Nowruz (Persian New Year)
    - Anniversary of revolution
    - Election cycles
    - Ashura/religious observances
```

### 4. Random Events

Probabilistic events that add variation.

```yaml
RANDOM_EVENT:
  type: random
  characteristics:
    - Probability-weighted
    - Conditioned on state (droughts more likely in certain conditions)
    - Add unpredictability
    - Never game-breaking; always recoverable
  examples:
    - Natural disasters (earthquake, drought)
    - International incidents
    - Personal events (illness, family)
    - Unexpected opportunities
```

### 5. Faction Events

Events generated by faction AI.

```yaml
FACTION_EVENT:
  type: faction
  characteristics:
    - Factions pursue their interests
    - React to player actions
    - Create dynamic world
    - Can be anticipated with intelligence
  examples:
    - IRGC economic initiative
    - Opposition announces unity platform
    - Foreign power makes diplomatic move
    - Civil society organizes demonstration
```

## Event Structure

### Complete Event Schema

```yaml
EVENT:
  # Identification
  id: string                    # Unique identifier
  title: string                 # Display title
  type: story | triggered | scheduled | random | faction

  # Narrative
  description: |
    Multi-line description of the situation.
    Can include dynamic text based on game state.
    Should be evocative, not just informative.

  context:                      # Background information
    what_happened: string       # The immediate situation
    why_it_matters: string      # Stakes and significance
    who_is_involved: list       # Key actors
    historical_parallel: string # Optional historical echo

  # Conditions
  triggers:                     # When does this fire?
    conditions: list[Condition] # All must be true
    probability: float          # 0-1, for random component
    priority: int               # Higher = checked first
    cooldown: duration          # Minimum time between occurrences
    max_occurrences: int        # How many times can this fire?

  # Timing
  timing:
    when: immediate | start_of_turn | end_of_turn | scheduled
    scheduled_date: date        # For scheduled events
    duration: turns             # How long event is active
    deadline: turns             # When decision must be made

  # Decisions
  decisions:
    - id: string
      label: string             # Button text
      description: string       # Hover/detail text

      # Requirements to see/choose this option
      requirements:
        - type: resource | relationship | state | trait
          condition: string
          message: string       # Why unavailable

      # What happens if chosen
      effects:
        immediate:              # Happens right away
          - type: string
            target: string
            value: number
        delayed:                # Happens later
          - type: string
            delay: turns
            probability: float
            target: string
            value: number
        narrative:              # Story consequences
          - triggers_event: string
            probability: float
          - unlocks: string
          - blocks: string

      # Faction reactions
      faction_reactions:
        faction_id:
          trust_change: number
          relationship_change: number
          triggers_response: string

      # Tone/character
      tone: pragmatic | principled | ruthless | compassionate
      advisor_opinions:
        advisor_type: string    # What they think

  # Visual/Audio
  presentation:
    image: string               # Event image
    mood: tense | hopeful | tragic | triumphant
    music_cue: string           # Optional music change

  # Learning
  learn_more:
    enabled: boolean
    topics:
      - title: string
        content: string
        sources: list[string]

  # Meta
  author_notes: string          # Design intent
  test_coverage: list[string]   # Which tests cover this
```

## Decision Design

### Principles for Good Decisions

```yaml
DECISION_PRINCIPLES:
  1_no_obvious_answer:
    description: |
      Every option should have a reasonable argument.
      If one choice is "obviously right," the decision is poorly designed.
    test: "Can I write a 200-word defense of each option?"

  2_meaningful_tradeoffs:
    description: |
      Choices should involve real sacrifice.
      Getting something should mean giving something up.
    test: "What does the player lose with each choice?"

  3_character_revealing:
    description: |
      Choices should reveal what the player values.
      Different players should choose differently.
    test: "Would a principled player choose differently than a pragmatic one?"

  4_consequence_bearing:
    description: |
      Choices should have lasting effects.
      The world should be different after this decision.
    test: "How does this choice affect events 10 turns later?"

  5_contextually_grounded:
    description: |
      Choices should make sense in Iranian political context.
      They should teach something about how this world works.
    test: "Does this reflect real political dynamics?"
```

### Decision Types

```yaml
DECISION_TYPES:
  dilemma:
    description: Two good things; you can only have one
    example: "Support workers (builds labor power) OR maintain coalition unity (republicans oppose)"

  sacrifice:
    description: Something good requires giving up something else good
    example: "Accept tainted alliance to gain power OR maintain purity but remain weak"

  risk_assessment:
    description: Certain small gain vs. uncertain large gain
    example: "Small safe reform OR risky revolutionary action"

  values_conflict:
    description: Two principles you hold conflict
    example: "Justice demands punishment BUT reconciliation requires amnesty"

  information_uncertainty:
    description: You must act without knowing everything
    example: "Is this defector genuine OR a regime plant?"

  timing:
    description: Act now (risky) or wait (might miss opportunity)
    example: "Strike while momentum exists OR wait for better conditions"
```

Events connect to form narrative arcs:


## Event Generation

### Dynamic Event System

Some events are generated rather than pre-authored:

```python
class EventGenerator:
    """
    Generates events based on game state.
    """

    def generate_faction_events(self, state):
        """
        Factions take actions based on their interests and state.
        """
        events = []

        for faction in state.factions:
            # Calculate faction priorities
            priorities = faction.calculate_priorities(state)

            # Check if faction will act
            if self.faction_will_act(faction, priorities, state):
                action = faction.choose_action(priorities, state)
                event = self.create_faction_event(faction, action, state)
                events.append(event)

        return events

    def generate_reaction_events(self, player_action, state):
        """
        Generate reactions to player actions.
        """
        events = []

        for faction in state.factions:
            reaction = faction.react_to(player_action, state)
            if reaction:
                event = self.create_reaction_event(faction, reaction)
                events.append(event)

        return events

    def generate_random_events(self, state):
        """
        Generate probabilistic events.
        """
        events = []

        for template in self.random_event_templates:
            # Check base conditions
            if not template.conditions_met(state):
                continue

            # Roll for probability
            if random.random() < template.probability(state):
                event = template.instantiate(state)
                events.append(event)

        return events
```

### Event Prioritization

When multiple events could fire:

```python
def prioritize_events(events, state):
    """
    Decide which events fire this turn and in what order.
    """
    # Score each event
    scored = []
    for event in events:
        score = calculate_event_priority(event, state)
        scored.append((event, score))

    # Sort by priority
    scored.sort(key=lambda x: x[1], reverse=True)

    # Apply limits
    max_events_per_turn = 3  # Configurable
    selected = []

    for event, score in scored:
        if len(selected) >= max_events_per_turn:
            break

        # Check for conflicts
        if not conflicts_with_selected(event, selected):
            selected.append(event)

    return selected
```

```yaml
        3. What are you willing to risk? Supporting workers
           brings you visibility - and vulnerability.

        4. How does this fit your broader strategy? Is this
           the right moment, or should you build more first?

        There are no right answers. Only tradeoffs.
```

## Implementation Details

### Event Engine

```python
class EventEngine:
    """
    Core event processing system.
    """

    def __init__(self, event_library, state):
        self.library = event_library
        self.state = state
        self.active_events = []
        self.event_history = []

    def process_turn(self):
        """
        Process all events for this turn.
        """
        # Gather candidate events
        candidates = []
        candidates.extend(self.get_triggered_events())
        candidates.extend(self.get_scheduled_events())
        candidates.extend(self.generate_faction_events())
        candidates.extend(self.generate_random_events())

        # Prioritize
        selected = self.prioritize_events(candidates)

        # Process each event
        results = []
        for event in selected:
            result = self.process_event(event)
            results.append(result)
            self.event_history.append(result)

        # Handle delayed effects from past events
        self.process_delayed_effects()

        return results

    def process_event(self, event):
        """
        Process a single event.
        """
        # Present to player
        presentation = self.present_event(event)

        # Wait for decision
        decision = yield EventPhase(event, presentation)

        # Apply effects
        effects = self.apply_effects(event, decision)

        # Record
        return EventResult(event, decision, effects)

    def apply_effects(self, event, decision):
        """
        Apply all effects of a decision.
        """
        decision_def = event.get_decision(decision.id)

        # Immediate effects
        for effect in decision_def.effects.immediate:
            self.state.apply_effect(effect)

        # Schedule delayed effects
        for effect in decision_def.effects.delayed:
            self.schedule_delayed_effect(effect)

        # Trigger narrative effects
        for narrative in decision_def.effects.narrative:
            self.process_narrative_effect(narrative)

        # Apply faction reactions
        for faction_id, reaction in decision_def.faction_reactions.items():
            self.apply_faction_reaction(faction_id, reaction)

        return self.state.get_changes()
```

## Quality Standards

### Event Review Checklist

Every event must pass:

```yaml
EVENT_REVIEW:
  narrative_quality:
    - [ ] Description is evocative, not just informative
    - [ ] Context explains why this matters
    - [ ] Language is appropriate for tone

  decision_quality:
    - [ ] All options have defensible arguments
    - [ ] No option is "obviously wrong"
    - [ ] Options reflect different values/strategies
    - [ ] Consequences are proportionate

  mechanical_quality:
    - [ ] Effects are balanced
    - [ ] No exploits or loopholes
    - [ ] Integrates with game systems properly

  educational_quality:
    - [ ] Teaches something about politics/Iran
    - [ ] Learn More content is accurate
    - [ ] Historical parallels are appropriate

  playtesting:
    - [ ] Multiple playtesters found decisions meaningful
    - [ ] Consequences felt fair and logical
    - [ ] No complaints of "that doesn't make sense"
```

## Conclusion

The Event System is how IRAN 14XX tells its story and presents its challenges. Well-designed events:

1. **Teach** - Every event communicates something about how politics works
2. **Challenge** - Every decision forces meaningful tradeoffs
3. **Connect** - Events chain together into compelling narratives
4. **Ground** - Events are rooted in real political dynamics
5. **Respect** - Players are treated as intelligent adults making hard choices

The goal is not to tell players what to do, but to create situations where they discover what they value and what they're willing to sacrifice.
